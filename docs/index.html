The KG Transformation

In Season 1, KG returned 2,400 tokens for a pricing query. By Season 4, predicate filters reduced that to 67 tokens — a 97.2% reduction. The same system that melted down on “Who are the active clients?” in S1 now delivers exact-fact answers at 15–67 tokens. Growth wasn’t just more data; it was smarter access patterns.

The Refinement Delta Story

GuildHouse’s value-add over raw QMD compressed across seasons. In S1, average refinement delta was +0.087. By S5, it was +0.036. That’s not failure — it’s success. The solo systems got better because GH was optimizing the pipeline they run on. The ceiling didn’t rise; the floor did.

5.0 — Series Verdict

The Racer Became the Router

GuildHouse won 19 of 24 quality races. It always produced the best or tied-for-best answer. But that was never the point.

The point was learning when to ask which system. Twenty-four races produced a routing table, not a scoreboard. The five rules in Section 4.4 are the deliverable — a production query router derived from empirical benchmarks.

Season 1 asked: “Which system wins?” Season 5 answered: “Wrong question. The right question is which system to send this query to.”

Race Director: "GuildHouse didn’t win the championship. It became the championship — the routing layer that decides who runs each race."

What 24 Races Proved

KG for exact facts — 15–67 tokens, perfect precision. When predicate filters match, nothing else comes close.
QMD for narrative — ~750 tokens for history, evolution stories, design rationale. The knowledge graph has nothing to say here.
MEM for recency — when QMD’s index is stale, MEM has yesterday’s notes. Temporal routing made this a formal system pattern.
GuildHouse for synthesis — when the query needs multiple systems combined, GuildHouse orchestrates. Entity discovery + vocabulary injection + refined search.
Optimizations improve the system, not just the optimizer — S5’s temporal routing lifted QMD standalone by +0.35. The router made the solo systems better.

Document Purpose

Complete benchmark report for GuildHouse — 24 head-to-head races across 5 seasons comparing four retrieval architectures: a knowledge graph (KG), a vector document store (QMD), auto-memory files (MEM), and GuildHouse (hybrid router). The series began as a benchmark competition and produced a production query routing table.

Key Findings (24 Races)
  • GuildHouse won quality in 19/24 races — always the best or tied-for-best answer
  • KG took the efficiency championship from QMD in Season 4, powered by predicate filters and fast-path (8 vs 11 cumulative, but KG led in S4-S5)
  • The 24 races produced a 5-rule routing table: exact fact → KG, broad entity → KG+QMD, narrative → QMD, relationship → QMD→KG backfill, recent → MEM→QMD
  • KG token usage on pricing queries dropped 97.2% (2,400 → 67) between S1 and S4
  • S5 optimizations improved the entire system, not just GuildHouse — temporal routing lifted QMD standalone by +0.35
  • GuildHouse graduated from benchmark racer to production query router
  • Season 3 results are classified due to “funny business”
The Routing Table (Production)
  • Exact fact (temporal, pricing, status) → KG + predicate filter (15-67 tok)
  • Broad entity (“What is X?”) → KG + QMD synthesis (~1,875 tok)
  • Narrative (history, evolution) → QMD alone (~750 tok)
  • Relationship (who, what connects) → QMD → KG entity discovery → refined QMD (~1,200 tok)
  • Recent activity → MEM → QMD (~1,100 tok)
